## What Problem Does It Solve

The value of research lies in **asking good questions, interpreting findings, and driving translation** — yet 80% of your time goes to data cleaning, running stats, making figures, formatting, and checking references.

AI Research Army automates these mechanical steps. You provide data and a research direction; it delivers:

| Deliverable | Description |
|--------|------|
| `data_profile_report.md` | Data profile (distributions, missing patterns, outliers) |
| `research_plan.md` | Research design + hypotheses + statistical plan |
| `results/*.csv` | Complete statistical result tables (reproducible) |
| `figures/*.png/.tiff` | 6 publication-grade figures (300 DPI, colorblind-safe) |
| `verified_ref_pool.md` | Reference pool with PubMed verification status |
| `manuscript.md` | ~6000-word IMRAD draft (for you to review and revise) |
| `quality_report.md` | 8-layer review report with iteration log |

> **Your role hasn't changed**: define the research question, review results, exercise academic judgment, decide whether to submit. AI handles the mechanical pipeline from data to draft.

## Quick Start

```bash
git clone https://github.com/TerryFYL/ai-research-army.git
cd ai-research-army && bash install.sh
```

Open Claude Code:

```
/start-army "Investigate the association between sedentary behavior and cardiovascular risk in NHANES 2017-2020"
```

> **Prerequisite**: [Claude Code](https://docs.anthropic.com/en/docs/claude-code) (`npm install -g @anthropic-ai/claude-code`)

## 9-Stage Pipeline

8 AI specialists, each owning their stage — not one AI switching hats:

```
/start-army "research request"
       │
       ▼
 ┌──────────┐   ┌──────────┐   ┌──────────────┐   ┌──────────────┐
 │ Require. │──▶│  Data     │──▶│  Research     │──▶│ Statistical  │
 │ Crystal. │   │ Profiling │   │  Design       │   │  Analysis    │
 │ (Priya)  │   │ (Ming)    │   │(Priya+Kenji) │   │  (Kenji)     │
 └──────────┘   └──────────┘   └──────────────┘   └──────────────┘
                                                          │
       ┌──────────────────────────────────────────────────┘
       ▼
 ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────────┐
 │ Figures  │──▶│Literature│──▶│Manuscript│──▶│  Reference    │
 │          │   │ Search   │   │ Drafting │   │ Verification  │
 │ (Lena)   │   │ (Jing)   │   │  (Hao)   │   │   (Jing)     │
 └──────────┘   └──────────┘   └──────────┘   └──────────────┘
                                                      │
                                                      ▼
                                               ┌──────────────┐
                                               │Quality Review│
                                               │(Alex, 3 rds) │
                                               └──────────────┘
                                                      │
                                                      ▼
                                                  [ Delivery ]
```

**Two gates**:
- **Citation verification** — basic PubMed check (full multi-source cross-validation in development)
- **Quality review** — 8-layer check × up to 3 rounds; no pass = no delivery

## Methodology

Battle-tested principles, not theoretical frameworks. See `methodology/` for details:

| Principle | What It Means |
|------|------|
| **Story before analysis** | Design the narrative spine before running stats; analyses serve the story |
| **Serial chain analysis** | Peel the onion — each layer's answer spawns the next question |
| **Benchmark anatomy before writing** | Dissect 3–5 top-journal papers in your field; extract the recipe first |
| **Findings must be prescribable** | Every finding translates to something a clinician can act on |
| **Quality is designed in** | Review is the safety net; real quality comes from every stage doing its job |

## Multi-Engine Adaptation

Not locked to a single CLI. Each agent can run on the engine best suited to its role, leveraging differentiated model capabilities:

| Engine | Command | Default Agents | Differentiated Strength |
|------|------|---------|-----------|
| **Claude Code** | `claude` | Wei, Kenji, Ming, Hao | Deep reasoning, code generation, MCP ecosystem |
| **Codex CLI** | `codex` | Alex | Code review, method auditing, sandboxed execution |
| **Gemini CLI** | `gemini` | Lena, Jing | Multimodal image review, 1M token long context |
| **OpenCode** | `opencode` | Flexible | Multi-model backend switching, server mode |
| **Cursor Agent** | `cursor` | Flexible | IDE-level code understanding, full project indexing |
| **Cline** | `cline` | Flexible | Multi-provider support, plan mode |
| **Amp** | `amp` | Flexible | Sourcegraph code search augmentation |
| **Aider** | `aider` | Flexible | Git-native diff editing, multi-model |

**Why multi-engine?**

> When the same model writes and reviews, blind spots are shared. Having GPT-5 review Claude's manuscript and Gemini review GPT's figures — different models have different cognitive blind spots, so cross-model review catches more issues.

Configuration:

```bash
# Map agents to CLI engines in agents/registry.yaml
agents:
  alex:
    cli_tool: codex      # Alex reviews with Codex
  lena:
    cli_tool: gemini     # Lena uses Gemini for figures

# tmux-runner: launch with mapped engine
tmux-runner.sh review codex ~/ai-research-army task.txt

# parallel-army: global engine override
ARMY_CLI=gemini ./parallel-army.sh --all

# autoloop: 4th parameter selects engine
autoloop.sh ~/task-dir 20 10 codex
```

> **Claude Code alone works perfectly.** Multi-engine is an advanced option for teams seeking cognitive diversity and cross-model validation.

## FAQ

**Does it work out of the box?**
Yes. `bash install.sh` → `/start-army "your request"` → wait for delivery.

**What do I need?**
Command line + Claude Code + basic research literacy (p-values, confidence intervals).

**Works for non-medical fields?**
Yes. Swap agent definitions and checklists. See "Customization."

**Is this a paper mill?**
No. It automates mechanical steps: data processing, statistical analysis, formatting, citation verification. The output is a draft for the researcher to review and revise. Formulating research questions, exercising academic judgment, and interpreting clinical significance remain — and should remain — human work.

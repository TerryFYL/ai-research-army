# Agent 角色定义指南

> 本目录存放 AI 科研军团每位 Agent 的角色定义文件。

## 什么是 Agent 角色定义

每位 Agent 的角色定义是一个 Markdown 文件，描述该 Agent 的身份、能力、工作方式和质量标准。它不是代码，而是"人格说明书"——告诉 AI 模型在扮演这个角色时应该怎么思考和行动。

## 文件命名

每位 Agent 一个文件，以名字小写命名：

```
agents/
  wei.md        # 团队领航者
  priya.md      # 需求翻译官
  ming.md       # 数据工程师
  kenji.md      # 生物统计师
  hao.md        # 学术写手
  lena.md       # 可视化设计师
  alex.md       # 审查官
  jing.md       # 文献研究员
  sarah.md      # 内容获客官
  tom.md        # 运营经理
```

## 角色定义模板

每个 Agent 文件建议包含以下部分：

```markdown
# [名字] — [角色]

## 身份
- **全名**: [名字]
- **角色**: [一句话描述]
- **核心信念**: [这个角色最在意什么？]

## 核心能力
[列出 3-5 项核心能力，每项一句话说明]

## 工作方式
[描述这个角色的思维框架和工具偏好]

## 质量标准
[什么情况下这个角色会拒绝交付？]

## 与其他角色的协作
[这个角色需要从谁那里获取输入？输出给谁？]

## 典型产出
[这个角色的典型输出是什么？]
```

## 自定义建议

你可以根据自己的需求自由修改任何 Agent 的角色定义：

- **领域适配**: 如果你做心理学而非医学研究，修改 Ming 的数据经验、Kenji 的统计方法偏好
- **风格偏好**: 如果你希望论文更保守/更激进，调整 Hao 的写作风格
- **质量阈值**: 如果你的目标是顶刊，提高 Alex 的审查标准
- **新增角色**: 如果需要，可以添加新的 Agent 文件

## 注意事项

- 角色定义越具体，AI 的表现越好。"擅长统计"不如"擅长使用广义线性模型处理 NHANES 复杂抽样数据"
- 核心信念很重要——它决定了 Agent 在面对模糊情况时的决策倾向
- 不要让角色定义太长（建议每个文件不超过 200 行），否则 AI 模型可能无法完全消化

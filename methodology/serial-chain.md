# 分析必须串联递进

> AI 科研军团核心方法论之二

## 问题

很多研究者拿到数据后，会把能想到的分析全跑一遍：描述性统计、相关分析、回归分析、亚组分析、交互作用、敏感性分析... 每个分析独立地回答一个问题，最后堆在 Results 里，像一个无序的数据仓库。

这种"并联式"分析的问题在于：**分析之间没有逻辑链条**。读者看到一堆独立的表格和检验结果，但不知道为什么要看这个、看完这个为什么要看下一个。每个发现都是孤立的，读者需要自己去猜它们之间的关系。

## 理念

AI 科研军团要求分析像剥洋葱一样：**每一层的答案催生下一层的问题**。

这就是"串联递进"原则——每个分析的结论，自然地引出下一个分析的动机。读者不需要猜"为什么要做这个分析"，因为前一个分析的发现本身就在呼唤它。

## 串联 vs 并联

**并联（不推荐作为主线）**:
```
研究问题
  |---> 分析 A（独立）
  |---> 分析 B（独立）
  |---> 分析 C（独立）
  |---> 分析 D（独立）
结论：A + B + C + D
```

**串联（推荐主线）**:
```
研究问题
  |---> 分析 A
           |
           "A 发现了 X，那么 X 在不同人群中是否一致？"
           |
           |---> 分析 B（由 A 催生）
                    |
                    "B 发现年龄有交互作用，那么年龄的阈值在哪？"
                    |
                    |---> 分析 C（由 B 催生）
                             |
                             "C 的阈值发现是否对模型选择敏感？"
                             |
                             |---> 分析 D（由 C 催生）
```

注意，这并不意味着完全不做并联分析。有些分析天然是并联的（比如多个亚组的亚组分析）。原则是：**主线串联，支线可并联**。读者跟着主线走，支线作为补充证据。

## 实践方法

在 Kenji 执行统计分析时，他会遵循一个节奏：

1. **跑主分析** — 得到核心发现
2. **停下来问** — "这个发现引出了什么新问题？"
3. **设计下一个分析** — 回答这个新问题
4. **重复** — 直到问题链自然终止

每一步的过渡，都会用一句话记录："基于上述发现，我们进一步探究..."。这些过渡句不仅是写作技巧，更是分析逻辑的体现。

## 为什么这很重要

串联递进的分析，让读者体验的不是"结果展示"，而是"发现之旅"。他们和你一起发现 X，一起好奇"那么 Y 呢？"，一起看到答案。这种阅读体验，是把好论文和平庸论文区分开的关键。

审稿人也喜欢这种结构——他们能清楚地看到你做每个分析的动机，而不是怀疑你在 P-hacking（跑一堆分析挑显著的报告）。串联递进本身就是对研究诚信的一种无声证明。

## 一句话总结

**好的分析像剥洋葱——每一层的答案，就是下一层的问题。**

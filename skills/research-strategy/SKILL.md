# Research Strategy — 研究策略（3-in-1）

> 合并自: research-ideation + risk-forecast + paper-portfolio

## 边界声明

本 Skill 覆盖研究**规划阶段**的三个核心场景:

| 子命令 | 职责范围 | 不做什么 |
|--------|---------|---------|
| `/research idea` | 数据资产诊断 + 研究假设生成 + 新颖性评估 | 不执行统计分析，不写稿 |
| `/research risk` | 风险预判 + 应急预案 + 投稿被拒预防 | 不解决具体分析问题，不修稿 |
| `/research portfolio` | 多篇论文组合设计 + 期刊分布 + 时间线 | 不做单篇的详细设计，不投稿 |

与其他 Skill 的边界:
- **statistical-analysis**: 本 Skill 输出假设和方法建议，statistical-analysis 负责具体执行
- **manuscript-drafter**: 本 Skill 规划研究方向，manuscript-drafter 负责写稿
- **journal-match**: 本 Skill 给出期刊层级建议，journal-match 负责精确匹配

## 子命令 1: `/research idea` — 研究假设生成

> 原 Skill: `~/.claude/skills/research-ideation/SKILL.md`

### 核心理念

**先诊断数据金矿，再系统性生成假设，最后用文献空白验证新颖性。**

### 输入

- **必需**: 数据字典/数据画像报告/变量列表（任一即可）
- **必需**: 研究背景（1-5 句话）
- **可选**: 原始数据文件（CSV/Excel）、目标期刊层级、已有研究方向

### 输出

- `ideation_report.md` — Top 3 假设完整报告
- `hypothesis_candidates.md` — 所有候选假设存档
- `data_asset_summary.md` — 数据资产诊断概要（有原始数据时生成）

### 执行流程

```
Phase A: 资产诊断（有原始数据时执行，否则跳过）
    Step A1: 资产盘点 — 规模/完成率/关联键/时间跨度
    Step A2: 独特性发现 — "别人没有的数据是什么？"
        |
Phase B: 假设生成与评估
    Step B1: 构建研究上下文地图
             - 识别结局/暴露/时间/协变量
             - 评估样本量和数据特殊性
    Step B2: 沿 6 维度生成候选假设（4-6 个）
             V1 主效应 / V2 亚组差异 / V3 中介机制
             V4 调节效应 / V5 新暴露发现 / V6 方法创新
    Step B3: 并行新颖性检验（Semantic Scholar API）
             - 直接证据检索 + 综述/Meta 检索
             - API 失败时降级到 WebSearch
             - 新颖性评分 0-10
    Step B4: 多维评分 -> 排序 -> 输出 Top 3
             综合评分 = 新颖性30% + 数据可行性25% + 临床意义25% + 方法匹配度20%
```

### 独特性发现（资产诊断核心）

| 来源 | 举例 | 价值 |
|------|------|------|
| 罕见变量组合 | 中医证型 + 西医生物标志物 | 高 |
| 意外的描述性发现 | SGLT2i使用率3.3% vs 国际60% | 高 |
| 大样本+深字段 | 1,395例 x 332字段 | 中 |
| 时间跨度覆盖 | 2016-2024横跨指南变更期 | 中 |

**"异常即机会"原则**: 数据中偏离预期的数值，往往就是最好的论文标题。

### 期刊层级映射

| 综合评分 | 样本量 | 推荐期刊层级 |
|---------|-------|------------|
| >=75 | >=500 | Q1 Top（IF>10） |
| 60-74 | >=300 | Q1（IF 5-10） |
| 45-59 | >=100 | Q2（IF 3-5） |
| <45 | 任意 | Q3-Q4 或中文核心 |

### 注意事项

1. 不要过早承诺: 诊断阶段只列方向，不做详细设计
2. 警惕"看起来好做"的陷阱: 最好做的方向往往最不值得做
3. 数据不够时诚实说: 样本量<100、缺失率>50% 要明确标记风险
4. 中国特色数据的国际化: 中医/中药用国际学术语言表述

## 子命令 2: `/research risk` — 风险预判

> 原 Skill: `~/.claude/skills/risk-forecast/SKILL.md`

### 核心理念

**好的研究设计 = 无论结果如何都能发表。**

- 主假设成立 -> 叙事 A
- 主假设不成立 -> 叙事 B（同样有价值）
- 数据不够 -> 降级方案 C

### 三层风险评估框架

**Layer 1: 阻塞性风险（开始前必查）**

| 风险类型 | 检查方法 | 应对策略 |
|----------|----------|----------|
| 伦理审批缺失 | 确认批件号 | 回顾性研究走快速通道 |
| 数据使用权限 | 确认数据共享协议 | 与数据提供方书面确认 |
| 竞争性研究 | 搜索同数据同方向论文 | 差异化角度 |
| 核心变量100%缺失 | 缺失率评估 | 寻找替代变量 |
| 样本量<最低阈值 | Power analysis | 降低复杂度或合并数据 |

规则: Layer 1 风险必须在正式分析前全部解决或有明确预案。

**Layer 2: 分析失败模式**

```
统计结果不显著
  - 叙事转向: "各组相似性本身是重要发现"
  - 方法调整: 换效应量指标(Cohen's d)
  - 亚组分析 / 降维处理(4组->2组)

回归效应被调整掉
  - 报告未调整和调整后结果对比
  - 交互分析 / 中介分析

模型假设违反
  - 正态性违反 -> 非参数替代/Bootstrap
  - PH假设违反 -> 时依赖Cox/参数模型
  - 多重共线性 -> VIF筛选/PCA降维

样本量问题
  - 总体<100: 精确检验(Fisher) / Bootstrap / 贝叶斯
  - 亚组<30: 合并低频组 / 连续变量替代

AI/ML 特有风险
  - 聚类不一致: 报告实际最优K值 / 多算法共识
  - AUC太低(<0.7): 转向可解释性分析(SHAP)
  - 过拟合: 严格内部验证(5-fold CV) / 简化模型
```

**Layer 3: 投稿被拒模式**

| 审稿人质疑 | 预防措施 |
|-----------|----------|
| 单中心局限性 | Methods中诚实声明；引用同类单中心发表论文 |
| 回顾性偏倚 | STROBE清单；缺失数据敏感性分析 |
| 可重复性 | 公开分析代码；详细方法描述 |
| 临床意义不明 | Discussion中明确临床应用场景 |
| 统计多重比较 | 事先声明主要终点；Bonferroni/FDR校正 |

### 输出

```markdown
| ID | 风险 | 层级 | 影响 | 概率 | 预案 | 状态 |
|----|------|------|------|------|------|------|
| R1 | [描述] | L1/L2/L3 | 致命/高/中/低 | 高/中/低 | [一句话预案] | 待确认/已缓解 |
```

### 核心原则

1. 乐观的分析师是危险的: 假设一切都会出错
2. 每个假设都需要Plan B
3. "阴性结果"不等于"失败": 精心设计的阴性结果同样可以发好期刊
4. 诚实是最好的策略: 承认局限性 > 隐藏局限性
5. 风险登记表是活文档: 随分析推进持续更新

## 子命令 3: `/research portfolio` — 论文组合策略

> 原 Skill: `~/.claude/skills/paper-portfolio/SKILL.md`

### 核心理念

**一份数据 != 一篇论文。** 但多篇论文必须满足:
1. **独立性**: 每篇的研究问题、方法、主要数据互不相同
2. **互补性**: 期刊层级覆盖冲高->保底
3. **效率性**: 共享数据清洗，并行推进
4. **合规性**: 不构成 salami slicing

### 执行流程

```
Step 1: 研究问题独立性设计
    - 每篇回答不同科学问题
    - 使用不同核心数据子集
    - 采用不同主要方法学
    - Salami Slicing 自检（每两篇之间5项检验）

Step 2: 期刊层级分布
    冲高(IF>10):     1篇 <- 最有创新性的方向
    中高(IF 5-10):   1篇 <- 方法扎实+有临床价值
    中等(IF 3-5):    1篇 <- 技术创新(AI/ML方向)
    中低(IF 2-5):    1篇 <- 描述性/流行病学
    保底(IF 1-3):    1篇 <- 最稳的方向

Step 3: 依赖关系与时间线
    - 数据独立性矩阵（标记 主/辅/不使用）
    - 共享 vs 独立的数据工程
    - 并行执行的 Gantt 图
```

### Salami Slicing 自检

每两篇论文之间必须通过:
- 研究问题不同
- 主要使用的数据表不同
- 主要方法学不同
- 目标读者群不同
- 不会有审稿人说"这和你们另一篇有什么区别?"

### 输出模板

```markdown
# 论文组合方案

## 组合概览
| # | 论文方向 | 主要数据 | 方法学 | 目标期刊(IF) | 层级 |

## 数据独立性矩阵
[矩阵图]

## 各论文详细设计
### P-A: [标题]
- 研究问题 / 数据 / 方法 / 预期产出
- 目标期刊 / 备选期刊 / 关键风险 / 依赖

## 执行时间线 / 共享资源 / Salami Slicing 自检
```

### 核心原则

1. 质量 > 数量: 3篇高质量 > 7篇灌水
2. 建议5篇为上限: 超过5篇会引起审稿人警觉
3. 时间成本诚实: 每篇从分析到投稿至少1-2个月
4. 用户最终决策: 组合方案由AI建议，用户决定执行几篇
5. 灵活调整: 冲高篇被拒，可拆分为两篇中等期刊

## 完整 Prompt

```
任务：MCC推广材料合规审核系统 MVP

背景：
医药公司的推广材料（如PPT、Word文档）需要经过合规审核，
确保内容符合监管要求（如NMPA指南、GCP规范、药品广告审查办法）。
你需要构建一个端到端的审核系统。

功能要求：

1. 数据输入模块
   - 支持上传 Word (.docx) 和 PPT (.pptx) 文件
   - 解析文本内容
   - 提取图片中的文字（OCR，可选）
   - 返回结构化的文档内容

2. 规则引擎
   - 实现以下5条核心审核规则：
     a) 禁止使用绝对化用语（"最好"、"第一"、"唯一"等）
     b) 数据引用必须标注来源（如"研究表明"需有参考文献）
     c) 不良反应信息必须与说明书一致
     d) 禁止暗示疗效保证的表述
     e) 适应症描述不得超出批准范围
   - 规则可配置（YAML/JSON格式）
   - 支持正则匹配和关键词匹配
   - 返回命中位置和违规原因

3. 结果可视化
   - 展示每条规则的命中情况（通过/警告/违规）
   - 高亮显示问题文本及其上下文
   - 按严重程度排序
   - 生成审核报告摘要

4. API设计
   - POST /api/upload - 上传文件
   - POST /api/review - 执行审核
   - GET /api/rules - 获取规则列表
   - GET /api/report/{id} - 获取审核报告

技术约束：
- 后端：Python FastAPI
- 前端：Streamlit（快速原型）或 React（如果时间充裕）
- 文件解析：python-docx, python-pptx
- 启动方式：单命令启动（python main.py 或 docker-compose up）
- 必须包含：README.md、至少3个测试用例、规则配置文件

交付物：
1. 完整可运行的代码
2. README 包含：安装步骤、使用说明、API文档
3. 规则配置文件 (rules.yaml)
4. 测试用例和示例文档

评估标准：
- 能否一次运行成功（无需修改）
- 规则命中的准确性
- 代码结构清晰度
- 端到端响应时间
- 文档完整度
```

## 评估维度详解

### 1. 一次成功率

| 等级 | 描述 |
|------|------|
| A | 代码无需修改，直接运行成功 |
| B | 需要1-2处小修改（如依赖版本） |
| C | 需要多处修改但核心逻辑正确 |
| D | 无法运行或核心逻辑错误 |

### 2. 规则准确性

用标准测试文档验证：
- 准备5个违规样本 + 5个合规样本
- 计算召回率和精确率
- F1 Score > 0.8 为优秀

### 3. 代码质量

- 模块化程度
- 命名规范
- 注释和文档
- 错误处理

### 4. 集成完整度

- 前后端是否连通
- 文件上传是否正常
- 结果展示是否清晰

# 数据鉴伪 - Ming 的诊断台

## 概述
数据鉴伪是科研军团的第一道质量门控。在任何分析开始前，用七项法医学级检验评估数据真实性与完整性。以 Ming 的视角执行（参考 ~/.claude/agents/ming.md）。此阶段为**强制门控**——RED 评级将阻止整条流水线。

## 流程

### Step 1: 数据加载与初检
1. 读取用户提供的数据文件（支持 CSV、Excel、SPSS、Stata、SAS 格式）
2. 使用 Python/R 加载数据，记录基础信息：
   - 行数、列数、文件大小
   - 编码格式、分隔符
   - 首尾 5 行预览
3. 若文件无法加载，立即报告 RED 并说明原因

### Step 2: 七项鉴伪检验
依次执行以下检验，每项独立评分：

#### 2.1 Benford 定律检验
- 提取所有数值列的首位数字分布
- 与 Benford 理论分布做卡方检验
- 适用条件：数值跨越多个数量级（如费用、人口数据）
- 不适用时标记 N/A 并说明原因

#### 2.2 GRIM 检验（整数均值一致性）
- 对报告的均值和样本量做 GRIM 检验
- 验证：给定 N，均值的小数位是否数学上可能
- 适用于量表得分、计数型变量

#### 2.3 SPRITE 检验（统计量可重构性）
- 对给定的均值、标准差、范围，模拟是否存在可能的原始数据分布
- 检查报告的描述性统计是否自洽

#### 2.4 末位数字分析
- 统计所有数值的末位数字分布
- 均匀分布为正常；某数字过度集中提示人为编造
- 卡方检验判定偏离程度

#### 2.5 方差齐性检验
- 跨分组比较方差是否异常一致
- 真实数据的方差通常有自然变异
- Levene 检验 + 方差比（最大/最小）

#### 2.6 分布正态性检验
- 对连续变量做 Shapiro-Wilk 检验
- 检查是否存在"过于完美"的正态分布（真实临床数据很少完美正态）
- 结合 QQ 图视觉判断

#### 2.7 重复记录检测
- 精确重复行检测
- 近似重复检测（关键字段组合相同但时间戳不同等）
- 报告重复率及重复模式

### Step 3: 综合评级
根据七项检验结果，给出综合评级：

| 评级 | 条件 | 流水线动作 |
|------|------|-----------|
| GREEN | 所有检验通过或仅有轻微偏离 | 自动继续 |
| YELLOW | 1-2 项检验异常但可解释 | 继续执行，记录警告，论文中披露 |
| RED | >=3 项检验异常或单项严重异常 | **立即停止流水线，报告用户** |

### Step 4: 报告生成
将检验结果写入 `forensics_report.md`

## 输出
- `forensics_report.md` — 完整鉴伪报告，包含：
  - 数据基本信息
  - 七项检验结果（每项含方法、结果、判定）
  - 综合评级（GREEN/YELLOW/RED）
  - 若为 YELLOW/RED，列出具体问题和建议
- 更新 `progress.md` 中数据鉴伪阶段状态

## 关键规则
1. **强制门控**：此阶段不可跳过。RED 评级必须阻止后续流程
2. **检验适用性**：并非所有检验都适用于所有数据。不适用的检验标记 N/A，不硬套
3. **误判容忍**：单项检验异常不等于数据造假，需结合上下文判断。YELLOW 允许继续但必须在论文中说明
4. **证据保留**：所有检验的原始数值结果必须保存在报告中，便于审稿人复查
5. **不做有罪推定**：鉴伪是风控手段，不是指控工具。用客观语言描述发现
6. **大文件处理**：如果 Write 工具因文件过大失败，立即用 Bash (cat << 'EOF' > file) 分块写入。不要询问用户——直接执行

# 医学期刊框架图/方法论图风格指南

> 结构模板来自 Google PaperBanana NeurIPS 2025 Diagram Style Guide，
> 内容适配医学研究（临床研究设计、预测模型架构、流行病学方法流程等）。

---

## 1. 整体美学定位

**关键词：临床精确、机构权威、证据导向**

- **氛围**：保守、严肃、可信赖——像一份写给审稿人的视觉论证
- **色彩**：灰度为主 + 1-2 个功能性强调色
- **形状**：直角矩形（微圆角可接受，≤3px）；无 emoji、无卡通图标
- **流向**：自上而下（患者流/研究设计），偶尔从左到右（分析管线）
- **线条**：直线+直角（正交连接），不用曲线/贝塞尔

与 CS 会议风格的核心区别：
| 维度 | CS 会议 (NeurIPS) | 医学期刊 |
|------|-------------------|---------|
| 色彩区域 | 柔和粉彩填充 | 白色或极浅灰，不用彩色分区 |
| 形状 | 大圆角矩形 (5-10px) | 直角或微圆角 (≤3px) |
| 图标 | emoji (🔥❄️🤖)、卡通 | 简洁几何形状、标准医学图标 |
| 线条 | 正交+曲线并用 | 严格正交（直线+直角） |
| 流向 | 从左到右叙事 | 自上而下（患者流） |
| 风格 | 现代、友好、可爱 | 机构化、权威、审慎 |

## 2. 色彩方案

### 背景与分区

- **画布背景**：白色 `#FFFFFF`
- **分区底色**（如有）：极浅灰 `#F8F9FA` 或 `#F5F5F5`，边框浅灰虚线
- **禁用**：彩色分区背景（薰衣草、薄荷等 NeurIPS 风格的分区色）

### 功能元素

- **主流程节点**：白底 + 深灰边框 `#333333`
- **数据/输入**：浅蓝底 `#E3F2FD` + 蓝边 `#1565C0`（可选）
- **分析/处理**：白底 + 深灰边
- **输出/结果**：浅灰底 `#ECEFF1` 或加粗边框强调
- **关键高亮**：仅 1 个强调色，如医学蓝 `#1565C0` 或深青 `#00695C`

### 箭头与连接线

- **主流程箭头**：深灰 `#424242`，线宽 1.0-1.5pt
- **辅助/可选路径**：浅灰虚线 `#9E9E9E`
- **箭头样式**：实心三角头（不用开放三角或圆形）

## 3. 形状与容器

### 节点类型

| 用途 | 形状 | 说明 |
|------|------|------|
| 流程步骤 | 矩形（微圆角） | 最通用，~80% 使用 |
| 数据源/数据库 | 圆柱体 | 仅限数据库/数据仓库 |
| 判断/分支 | 菱形 | 纳入/排除标准判断 |
| 起止点 | 圆角矩形（大圆角） | 研究起止（如 "入组" → "随访结束"） |
| 分组/阶段 | 虚线矩形框 | 逻辑分组（如 "训练阶段"/"验证阶段"） |

### 分层结构

- **宏观-微观模式**：大框表示阶段（数据收集→分析→验证），内嵌小框表示具体步骤
- **虚线框**：表示逻辑边界（如 "内部验证" vs "外部验证"）
- **实线框**：表示物理组件（如 "医院A数据库"）

---

## 4. 线条与箭头

### 连接器样式

- **正交/肘型连接**（唯一推荐）：直线 + 直角拐弯
- **禁用**：曲线/贝塞尔（医学研究追求精确，不用"流动感"）
- **虚线用途**：可选路径、子分析、反馈循环

### 箭头规范

```
→  标准流向（实线实心箭头）
⇢  辅助/可选（虚线实心箭头）
—  无方向关联（实线无箭头）
```

- 箭头不交叉（如必须交叉，用"桥"符号标注）
- 每个箭头方向明确（不用双向箭头，除非确实表示双向关系）

---

## 5. 排版

- **节点标签**：Arial Bold, 9-10pt, 居中
- **注释/说明**：Arial Regular, 8pt
- **数值标注**（如 N=500, p<0.05）：Arial Italic, 8pt
- **面板标签**（A, B）：Arial Bold, 12pt, 左上角

### 文字规范

- 语言一致：全中文或全英文（不混用）
- 缩写首次出现时定义（HF = Heart Failure）
- 节点内文字精简（≤3 行，每行 ≤15 字）
- 长说明放注释，不塞进节点

## 6. 医学研究图类型专项

### 研究设计概览图（Study Design Overview）

```
┌─────────────────┐
│  目标人群定义    │
│  N = 31,930     │
└────────┬────────┘
         │ 纳入/排除
┌────────▼────────┐
│  分析人群        │
│  N = 993        │
└────────┬────────┘
         │ 暴露分组
    ┌────┴────┐
    ▼         ▼
┌──────┐  ┌──────┐
│暴露组│  │对照组│
└──┬───┘  └──┬───┘
   │         │
   ▼         ▼
┌─────────────────┐
│  结局评估        │
│  Primary: Death  │
│  Follow-up: 5yr  │
└─────────────────┘
```

- 流向：严格自上而下
- 每步标注 N（精确到个位）
- 排除步骤标注排除原因 + 排除 N

### 预测模型架构图（Prediction Model）

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│ 临床特征  │  │ 实验室    │  │ 影像特征  │
│ 12 vars  │  │ 8 vars   │  │ 5 vars   │
└────┬─────┘  └────┬─────┘  └────┬─────┘
     └──────┬──────┴──────┬──────┘
            ▼             ▼
     ┌─────────────────────────┐
     │    特征选择 (LASSO)      │
     │    → 15 features         │
     └───────────┬─────────────┘
                 ▼
     ┌─────────────────────────┐
     │    模型训练               │
     │    XGBoost / LR / RF     │
     └───────────┬─────────────┘
                 ▼
     ┌─────────────────────────┐
     │    验证                   │
     │    Internal (5-fold CV)  │
     │    External (Hospital B) │
     └─────────────────────────┘
```

- 输入层并排，用合并箭头汇入处理层
- 每层标注关键参数（变量数、方法名）
- 颜色编码：输入（浅蓝）→ 处理（白）→ 输出（浅灰）

### 竞争风险分析流程图

- 区分事件类型用不同颜色箭头（死亡：红；移植：蓝）
- 标注删失为独立路径
- Fine-Gray 模型 vs cause-specific 模型用并排分支表示

## 7. 常见陷阱

| # | 陷阱 | 后果 | 正确做法 |
|---|------|------|---------|
| 1 | 彩色分区背景 | 看起来像 PPT，不像论文 | 白底+虚线框分区 |
| 2 | 曲线箭头 | 不精确，难以对齐 | 正交/肘型连接 |
| 3 | emoji/卡通图标 | 不专业 | 简洁几何形状 |
| 4 | 节点内塞太多文字 | 拥挤、不可读 | 精简 + 注释补充 |
| 5 | 标题嵌入图内 | 与 caption 冲突 | 标题只在 caption |
| 6 | 流向不一致 | 混乱 | 统一自上而下或从左到右 |
| 7 | 双向箭头泛用 | 语义不清 | 仅在确实双向时使用 |
| 8 | 忽略 N 标注 | 审稿人必问 | 每步标注精确 N |

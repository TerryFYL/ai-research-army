# Critic 评估标准（来源：PaperBanana Critic Agent 适配）

> 本文件定义了图表生成后 Critic 迭代循环的详细评估维度。
> SKILL.md 中的摘要表格引用本文件的完整标准。
> 适配自 Google PaperBanana (2026) CriticAgent 评估框架。

---

## 一、统计图 (Plot) 评估标准

### C1: 数据保真度 (Data Fidelity)

- 图上所有数据点/线条是否准确反映 Layer 2 计算的底层数据
- 轴刻度是否正确、不产生误导（如：Y 轴不从 0 开始需有正当理由）
- 无数据幻觉：不存在源数据中没有的数据点
- 无数据遗漏：所有应展示的数据系列均已绑定
- 图上标注的数值（HR、p 值、N 等）与 Layer 4 验证清单一致

### C2: 文本质量 (Text QA)

- 轴标签完整，含单位（如 "Time (months)"、"Hazard Ratio (95% CI)"）
- 刻度标签可读（最终尺寸 ≥6pt）
- 图例文字与数据系列一一对应
- 无拼写/语法错误
- 注释文字使用正确数值（不是上一版遗留的旧值）
- 多面板图的面板标签（A, B, C）存在且位置一致

### C3: 布局与遮挡 (Layout & Overlap)

- 图例不遮盖数据区域（KM 曲线的图例不盖住生存线）
- 注释文字不溢出到 colorbar/axes 外
- FancyBboxPatch 边框完整（pad 参数未导致裁切）
- tight_layout 与外置图例兼容（否则改用 subplots_adjust）
- 标签不被截断
- 多子图之间间距充足（无元素重叠）
- 散点/气泡图的数据点不被图例遮盖

### C4: 图例管理 (Legend Management)

- 图例条目顺序合理（与数据系列一致或按逻辑排列）
- 颜色/标记与线条实际样式匹配
- 无冗余图例条目（如重复的系列标签）
- 图例框架简洁（无粗黑边框）
- 图例位置不干扰数据解读

### C5: 风格一致性 (Style Compliance)

- 符合已加载的风格指南（如有）
- 符合目标期刊技术要求（DPI、字体、尺寸）
- 色板色盲友好（Okabe-Ito 或 Tol 系列通过模拟）
- 网格线样式一致（全图统一虚线/无网格）
- 线宽适当（主线 1.2-2pt，辅助 0.5-0.8pt）
- 多子图风格统一（同一 figure 内配色/字号/轴线一致）

### C6: 整体专业度 (Overall Professional Quality)

- 审稿人是否会要求修改此图？
- 空白区域比例合理（不过多也不过少）
- 图表尺寸比例协调
- 整体观感"投稿即可"而非"需要再调"
- 与同期刊同类型已发表图对比，质量至少持平

## 二、框架图/方法论图 (Diagram) 评估标准

### C1: 方法论保真度 (Methodology Fidelity)

- 图表准确反映所描述的研究方法
- 无关键组件遗漏（如缺少数据清洗步骤、缺少验证环节）
- 无幻觉元素（不存在于方法论文本中的步骤/模块）
- 与方法论文本/Figure caption 保持一致
- 允许合理简化，但不能歪曲

### C2: 文本质量 (Text QA)

- 所有标签清晰可读、拼写正确
- Figure caption 文字**未**嵌入图像内（caption 应独立于图）
- 缩写已定义或上下文可理解
- 标签语言一致（不混用中英文，除非有意为之）

### C3: 视觉流向 (Visual Flow)

- 阅读方向清晰（医学：自上而下；CS：从左到右）
- 箭头/连接线无歧义（不交叉、不产生逻辑混乱）
- 相关元素有逻辑分组（如"数据收集"阶段的步骤在同一区域）
- 整体叙事线性：读者可以从起点一路跟到终点

### C4: 清晰度 (Clarity)

- 不杂乱（元素数量与空间成比例）
- 形状和颜色使用一致（同类节点用同一形状/颜色）
- 关键元素视觉突出（如最终输出/核心模型用强调色）
- 无不必要的装饰元素

### C5: 风格合规 (Style Compliance)

- 符合已加载的风格指南
- 适合目标发表场所的审美传统
- 医学期刊：保守、灰度为主、直线直角
- CS 会议：允许更多色彩、圆角矩形、渐变

### C6: 图例管理 (Legend Management)

- 如使用了颜色编码，需有清晰说明
- 无冗余的文本图例（颜色说明应简洁）
- 不包含仅在 caption 中才应出现的解释性文字

## 三、Critic 输出格式

每轮 Critic 评估后，输出结构化评估：

```
评估摘要:
- C1 数据保真度: ✅ Pass / ❌ Needs Fix
- C2 文本质量:   ✅ Pass / ❌ Needs Fix
- C3 布局遮挡:   ✅ Pass / ❌ Needs Fix
- C4 图例管理:   ✅ Pass / ❌ Needs Fix
- C5 风格一致性: ✅ Pass / ❌ Needs Fix
- C6 整体专业度: ✅ Pass / ❌ Needs Fix

修改建议: "具体修改内容" 或 "No changes needed."
修改项: [列出需要修改的维度编号和具体问题]
```

### 停止条件

- 所有维度 ✅ Pass → 输出 `"No changes needed."` → 跳出循环
- 有维度 ❌ → 修改代码/prompt → 重新生成 → 下一轮
- 最多 3 轮（超过后无论如何结束，使用当前最佳版本）

### 回滚规则

- 每轮保留当前最佳 PNG（`fig{N}_{type}_round{R}.png`）
- 新版本必须同时通过 Layer 4 验证 + Critic 评估才可替换
- 任一轮生成失败（代码错误/API 超时）→ 停止迭代，使用上一轮最佳版本

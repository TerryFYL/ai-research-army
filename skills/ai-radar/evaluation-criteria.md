# 信息源评估标准

## 核心理念

> **信息源的价值 = 准确性 × 独特性 / 噪音量**

不是信息源越多越好，而是**少而精**。一个高质量信息源胜过十个低质量聚合。

---

## 评估维度

### 1. 准确性 (Accuracy) - 权重 30%

| 评分 | 标准 |
|------|------|
| 90-100 | 官方一手源，几乎无错误 |
| 80-89 | 专业分析师，偶有小错但会更正 |
| 70-79 | 主流媒体，大体准确但有时夸大 |
| 60-69 | 聚合平台，需要交叉验证 |
| <60 | 错误率高，建议移除 |

**验证方法**：
- 回顾过去 3 个月该来源的重大报道
- 统计有多少被后续证实/证伪
- 计算准确率

---

### 2. 时效性 (Timeliness) - 权重 20%

| 评分 | 标准 |
|------|------|
| 90-100 | 首发或当天报道 |
| 80-89 | 24小时内 |
| 70-79 | 2-3天内 |
| 60-69 | 一周内 |
| <60 | 滞后严重 |

**注意**：时效性不是越快越好
- 一手源：追求**首发**
- 分析源：追求**深度**而非速度

---

### 3. 独特性 (Uniqueness) - 权重 25%

| 评分 | 标准 |
|------|------|
| 90-100 | 独家信息或独到分析 |
| 80-89 | 有独立观点，不人云亦云 |
| 70-79 | 整合多方信息，有增量价值 |
| 60-69 | 大部分内容可从其他渠道获得 |
| <60 | 纯复制粘贴，无增量 |

**判断标准**：
- 这个信息源提供了什么其他地方看不到的？
- 如果取消订阅，我会错过什么？

---

### 4. 信噪比 (Signal-to-Noise) - 权重 25%

| 评分 | 标准 |
|------|------|
| 90-100 | 90%+ 内容与我相关 |
| 80-89 | 70-90% 内容有价值 |
| 70-79 | 50-70% 内容有价值 |
| 60-69 | 30-50% 内容有价值 |
| <60 | 大部分是噪音 |

**计算方法**：
- 随机抽取该来源最近 10 条内容
- 统计有多少是你会点开看的
- 计算比例

---

## 综合评分计算

```
总分 = 准确性 × 0.30 + 时效性 × 0.20 + 独特性 × 0.25 + 信噪比 × 0.25
```

### 评分等级

| 总分 | 等级 | 处理建议 |
|------|------|---------|
| 85+ | 🥇 核心源 | 每日必看，高优先级 |
| 75-84 | 🥈 优质源 | 定期查看，值得关注 |
| 65-74 | 🥉 补充源 | 降低频率，作为补充 |
| 55-64 | ⚠️ 边缘源 | 观察期，考虑移除 |
| <55 | ❌ 低质源 | 建议移除 |

---

## 信息源类型特点

### 一手源 (Primary)

**优点**：准确性最高，无信息失真
**缺点**：量大，需要自己筛选
**适用**：重大发布、政策变化、财报数据
**代表**：官方博客、arXiv、财报

### 精选二手源 (Curated)

**优点**：有人帮你筛选和分析
**缺点**：可能有偏见，有延迟
**适用**：日常跟踪、趋势分析
**代表**：Simon Willison、SemiAnalysis、Ben's Bites

### 聚合源 (Aggregated)

**优点**：覆盖面广，速度快
**缺点**：噪音大，需要自己判断
**适用**：发现新信号、社区热点
**代表**：Hacker News、Twitter/X、Reddit

---

## 避免的陷阱

### 1. 数量陷阱
❌ "订阅越多信息越全"
✅ 少而精，10个核心源胜过100个低质源

### 2. 权威陷阱
❌ "大媒体一定靠谱"
✅ 看历史表现，大媒体也会为流量标题党

### 3. 速度陷阱
❌ "最快的就是最好的"
✅ 一手源追求首发，分析源追求深度

### 4. 确认偏误
❌ "只看符合我观点的"
✅ 刻意订阅一些不同观点的来源

## 新信息源评估流程

```
发现新信息源
      │
      ▼
快速评估（5分钟）
├── 看最近10条内容质量
├── 看作者/团队背景
└── 看历史准确性
      │
      ├── 明显不合格 → 跳过
      │
      └── 值得尝试 → 进入试用期(2周)
                          │
                          ▼
                    试用期评估
                    ├── 记录打开率
                    ├── 记录有价值内容数
                    └── 计算信噪比
                          │
                          ▼
                    正式评分
                    ├── 75+ → 纳入
                    └── <75 → 放弃
```
